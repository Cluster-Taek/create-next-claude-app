# Spec 문서 템플릿

## 파일 헤더

```markdown
# 📋 {도메인} {기능} 요구사항 명세서
```

예: `# 📋 사용자 목록 페이지 요구사항 명세서`

## 표준 섹션 구조

### 1. 페이지 개요 (필수)

```markdown
## 1. 페이지 개요

- **목적:** {한 문장으로 요약}
- **진입 경로:** {어디서 어떻게 도달하는지}
- **특이사항:** {보안/성능/UX 제약 — 없으면 생략}
```

### 2. 영역별 상세 (필수, N개 반복)

UI를 큰 영역(Header, 검색 영역, 리스트, 팝업 등)으로 나눠 섹션을 만든다.

```markdown
## 2. [{영역명}]

### 2.1 {서브 영역}

- **{요소명}:** {설명 및 동작}
- **{요소명}:**
  - **동작:** {클릭/입력 시 동작}
  - **결과:** {결과 상태/데이터 변화}
```

표 형식이 더 명확한 경우(필드 제어, 컬럼 정의 등):

```markdown
### 2.2 {그리드/필드 제어}

| 필드명       | 입력 방식 및 제어 로직                          |
| :----------- | :---------------------------------------------- |
| **사용자명** | 텍스트 입력 (최대 50자)                         |
| **권한**     | 드롭다운 선택 (관리자, 일반, 게스트)            |
| **이메일**   | IF 권한 = "관리자": 필수 / ELSE: 선택           |
```

### 3. 비즈니스 로직 (조건/상태/트리거가 있을 때)

```markdown
## 3. 핵심 비즈니스 로직 및 상태 정의

### 3.1 {상태/엔티티명} 정의

| 상태명     | 정의              | 발생 트리거          | 비고          |
| :--------- | :---------------- | :------------------- | :------------ |
| **활성**   | {정의}            | {트리거}             | -             |
| **휴면**   | {정의}            | {트리거}             | -             |
| **차단**   | {정의}            | {트리거}             | -             |

### 3.2 {매핑/연동 규칙}

- **트리거:** {조건}
- **연동 동작:**
  1. {단계 1}
  2. {단계 2}
```

### 4. 페이지 이동 (네비게이션이 있을 때)

```markdown
## 4. 페이지 이동 (Navigation)

- **{액션}:** {이동 대상} ({전달 상태})
```

## 작성 규칙

- 제목은 `📋` 이모지로 시작 (사용자가 명시적으로 이모지 제외 요청 시 생략)
- 각 섹션은 `##` 사용, 서브섹션은 `###`
- 동작/결과는 **굵게** 표시한 라벨로 구분
- 분기 조건은 `IF / ELSE / CASE` 키워드 사용 (예: `IF 권한 = "관리자": 수정 가능 ELSE: Read-only`)
- 상태/매핑은 표로 작성하면 가독성 향상
- 영문 키(`status`, `workStatus` 등)는 백틱으로 감싸기
- 한글로 자연스럽게 작성, 코드 키워드만 영문 유지

## 파일 명명 및 위치

- 파일명: `{도메인-소문자}-{기능-소문자}.spec.md`
  - 예: `user-list.spec.md`, `user-detail.spec.md`, `user-register.spec.md`
- 위치: `docs/spec/` (없으면 사용자에게 확인 후 생성)
- 버전 관리: 파일 내부에 `Ver_YYMMDD` 표기 불필요 (git 히스토리로 관리)

## 체크리스트

Spec 작성 후 다음을 확인:

- [ ] 페이지 개요에 목적과 진입 경로가 명시됨
- [ ] 모든 UI 영역이 섹션으로 분리됨
- [ ] 분기/조건이 IF-ELSE/CASE로 명시됨
- [ ] 상태값이 있으면 정의표로 정리됨
- [ ] 외부 연동(다른 도메인 영향)이 별도 섹션으로 분리됨
- [ ] 추측한 내용이 없으며, 모호한 부분은 `TBD`로 표시됨
